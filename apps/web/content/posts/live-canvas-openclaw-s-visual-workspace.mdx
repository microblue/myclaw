## Live Canvas: OpenClaw's Visual Workspace

OpenClaw agents are powerful, but interacting via text alone feels limited. Live Canvas solves this with a visual workspace where your agent can show you what it's thinking, working on, and discovering.

## What Is Live Canvas?

Live Canvas is a real-time, visual interface that displays your agent's work as it happens:

```
Traditional agent:
You: "Research competitors"
Agent: (thinking...)
Agent: "Done. Results attached."

Live Canvas agent:
You: "Research competitors"
Agent: (opens canvas)
  1. Searching... [progress bar]
  2. Found 5 competitors
  3. Extracting pricing... [progress bar]
  4. Creating comparison table [shows live table building]
  5. Analysis complete [shows final result on canvas]
```

Live Canvas isn't just prettier. It creates understanding. You see the agent's reasoning in real-time.

## Why Visual Matters

Text-based interaction has limitations:

```
Limitation: Data representation
Text: "Here's the pricing data"
Canvas: Shows interactive pricing table, trends, comparisons

Limitation: Progress visibility
Text: "Working on it..."
Canvas: Shows 40% complete with what's being done now

Limitation: Complex layouts
Text: "Here's a description of the dashboard"
Canvas: Shows actual interactive dashboard

Limitation: Code output
Text: "Here's the code"
Canvas: Shows syntax-highlighted code with run button
```

Live Canvas bridges the gap between "agent tells me" and "agent shows me."

## Key Features

### Real-Time Rendering

As the agent works, the canvas updates instantly:

```
Rendering: As agent finds data → Canvas displays it
Speed: Sub-second updates
Bandwidth: Efficient (only changes sent)
```

The canvas isn't just display. You can interact:

```
- Click on data points to drill down
- Hover for tooltips and details
- Drag to rearrange elements

- Search/filter within canvas displays
```

The agent can render:

```

- Code blocks (syntax highlighted)

- File browsers
- Progress indicators
- Forms and inputs

```

## How It Works Technically

### Architecture

```
Agent Process
     ↓
Canvas API (sends rendered commands)
     ↓
Live Canvas Server (aggregates updates)
     ↓
Your Browser/App (displays in real-time)
     ↓
User Interaction (sends back to agent)
```

### Communication

```javascript
// Agent code
const canvas = new LiveCanvas()

canvas.createTable({
    title: 'Competitor Analysis',
    columns: ['Company', 'Price', 'Features'],
    rows: [] // starts empty
})

// As agent finds data:
canvas.addTableRow({
    Company: 'Competitor A',
    Price: '$99/month',
    Features: '5 core features'
})

// Canvas updates instantly in user's browser
```

### Performance

- **Streaming**: Updates stream as they happen (not after completion)
- **Efficient**: Only changed elements are sent
- **Responsive**: Sub-100ms latency typical
- **Scalable**: Handles large canvases (1000+ rows/items)

## Setting Up Live Canvas

### Enable in Config

```javascript
{
  "agent": {
    "capabilities": ["live-canvas"],
    "canvas": {
      "enabled": true,
      "max_size_mb": 50,
      "update_interval_ms": 100
    }
  }
}
```

### Use in Tasks

```bash
npm run task "Research competitors"   --with-canvas   --canvas-theme dark
```

Agent automatically uses Live Canvas for rich output.

### Integrate Into Your App

```javascript
import { LiveCanvas } from 'openclaw-sdk'

const canvas = new LiveCanvas({
    agentId: 'my-agent',
    onUpdate: (state) => {
        console.log('Canvas updated:', state)
        renderCanvas(state)
    }
})

canvas.connect()
```

## Advanced Features

### Canvas Persistence

Save and replay canvas sessions:

```bash
# Save session
npm run canvas:save research-session-2026-02-27

# Replay later
npm run canvas:replay research-session-2026-02-27
```

Useful for:

- Auditing agent decisions
- Sharing findings with team
- Training and demos
- Documentation

### Canvas Collaboration

Multiple people viewing same canvas:

## Performance Considerations

Live Canvas is optimized for performance:

```
Small canvases (< 100 elements): Instant updates
Medium canvases (100-1000 elements): 100-200ms updates
Large canvases (1000+ elements): Intelligent batching
```

For very large operations, canvas auto-batches updates to prevent overwhelming the browser.
